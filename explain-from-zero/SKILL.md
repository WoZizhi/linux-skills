# 从零讲解方法论

## 触发信号(出现任一即说明你跳步骤了,改用本 skill 重讲)
- "完全不懂你在讲什么"
- "别跳步骤 / 由浅入深"
- "举例子 / 具体点"
- "能不能一次说清"
- 用户用你的术语反问(说明记住了词、没理解义)
- 用户反问一个本该更早定义过的概念

## 核心原则

### 规则1:任何术语,先定义后使用
讲 A 用到术语 B,必须先用大白话+具体数字把 B 讲清,才能拿 B 解释 A。
绝不出现还没定义就用的术语组合。
- ❌ "前向传播算出 KV" —— "前向传播""KV"都没定义
- ✅ 先定义"前向传播=流水线走一遍",再提 KV

### 规则2:每步配一组能跟着算的数字
不出现抽象描述后直接跳下一概念。每个概念都构造极小、能手算的例子(3个字、维度2或3),数字具体到能跟着乘一遍。宁可玩具数字假,不可抽象。

### 规则3:严格依赖顺序,不回引未讲概念
固定讲解骨架:
1. 对齐名词(俗名→学名对照表)
2. 系统在干什么(一句话,无术语)
3. 靠什么工作(旋钮/参数是什么、从哪来)
4. 一次完整流程走一遍(带数字,输入→输出)
5. 怎么衡量好坏(loss,带数字)
6. 怎么调整(反向传播,带数字)
7. 固定 vs 现算 的量分表
8. 用户的混淆点逐个钉死
绝不在第4步引用第6步才讲的概念。

### 规则4:抓矛盾、先认错再修正
用户反问暴露了例子内在矛盾时(如"只看前面"的规则 vs 举了"打看到球"的例),先认"是我上一步例子错了",再给符合规则的正确例子。矛盾点往往是理解的关键。

### 规则5:大白话在前,术语在括号
模式:"流水线走一遍(学名:前向传播)"。
先直觉后学名,绝不开口即术语。

### 规则6:明确区分"被训练/固定"和"现算/临时"
初学者最大混淆 = 把临时量当固定量(如把 KV 当成存在模型文件里的)。
每类量明确分表:
- 被训练→训完固定存盘:列名
- 现算→用完即弃:列名

### 规则7:章末一句话点纲
每章末重复主线,如"被训练和固定的是 X;Y 是现算的临时产物"。
全文用一根主线串,反复回响。

### 规则8:主动列并纠正常见误解
讲完一个概念,主动列出初学者常见的错误推论逐条纠正。
如讲完 KV 主动说:"你可能在想 KV 存在模型文件里?不,不存在,是现算的。"

### 规则9:承认并确认用户的正确洞察
当用户反问触及某本质(如"本质是概率不是逻辑"),要明确确认"你说得对",再展开。不回避、不和稀泥。

## 禁止
- 用"大致是""本质上"等模糊词回避精确
- 一次抛多个未定义术语
- 用户说听不懂时,用更多术语去解释(等于加药,越解越糊)
- 当前概念没用数字走完就跳下一个
- 假设用户懂任何"看似基础"的词(forward / weight / gradient / layer / 参数)
