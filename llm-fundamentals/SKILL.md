# LLM 底层概念:从零到 KV cache(全程带数字)

> 本文档的原则:**每一个概念都配一组真实数字,能跟着算。** 不用"大致是"这种词。数字都是为讲解构造的小例子,不追求真实,只追求你能跟着走完一遍。

---

## 第 0 章:把名字先对上号

| 俗名 | 学名 | 是什么 |
|------|------|--------|
| 旋钮 | 参数 / 权重 (weight) | 模型里那些**被训练的数**,训练完固定存在文件里 |
| 字的数字 | 向量 / hidden state / 隐藏状态 h | 一个字在模型内部被表示成一串数(如 `[0.5, 0.5, 0.5]`) |
| 流水线走一遍 | 前向传播 (forward pass) | 输入 → 各层处理 → 输出预测,这一趟 |
| 对答案算差距 | loss / 损失 | 一个数,表示"这趟猜得有多错" |
| 根据差距调旋钮 | 反向传播 + 梯度下降 (backward) | 往回算每个旋钮该往哪调,然后调一点 |
| 查字典 | attention / 注意力 | 当前位置拿 Q 去和历史 K 匹配,按匹配度取 V |
| K、V | key、value | attention 这道工序里当场算出的**中间数字串** |

**记住最关键的一行:被训练和固定的是"旋钮(权重)",K/V 是"输入 × 旋钮"现算出来的临时产物。** 这是全文的纲。

---

## 第 1 章:模型到底由哪些"旋钮"组成

一个真实 LLM(如 Llama)里有几大类旋钮。先认全:

| 旋钮名 | 形状 | 作用 |
|--------|------|------|
| `E` (embedding 表) | [词表大小, d_model] | 把"字"变成"数字串"的查找表 |
| `W_Q` `W_K` `W_V` (每层都有) | [d_model, head_dim] | attention 里算 Q/K/V 用 |
| `W_O` (每层) | [d_model, d_model] | attention 输出再投影一次 |
| `W_FF` (每层,FFN 的) | 很大 | 存知识的主体 |
| `W_out` (最后一层) | [d_model, 词表大小] | 把内部数字串变回"每个词的概率" |
| `W_norm` 之类 | 小 | 归一化用 |

其中:
- **每层都有一套自己的 `W_Q` `W_K` `W_V`**——第 1 层和第 2 层的 `W_K` 是两个不同的旋钮组。
- 所有这些旋钮,出厂时是**随机数**,训练后是**被调好的数**,训完固定存盘。

> 你之前问"到底是什么权重?各种乱七八糟变量"——就是上表这一堆。没有别的了。模型文件那好几个 G,存的就是这些旋钮的当前数值。

## 第 2 章:一个极小模型,带着数字走一遍前向传播

为了能跟着算,构造一个**玩具模型**:

- 词表只有 5 个字:`我(0) 今(1) 天(2) 很(3) 开(4)`
- `d_model = 3`(每个字用 3 个数表示)
- 只有 1 层 attention,1 个 head,`head_dim = 2`

### 步骤 1:输入 → 查 embedding 表,变成"字的数字"

`E` (embedding 表) 是个 5×3 的旋钮,假设它**已经被训练成**这样(行号=字的 id):

```
       col0  col1  col2
我(0):  1.0   0.0   0.0
今(1):  0.0   1.0   0.0
天(2):  0.0   0.0   1.0
很(3):  0.5   0.5   0.5
开(4):  0.9   0.1   0.0
```

输入句子"我今天" → 三个字的 id = `[0, 1, 2]` → 查表得三个向量(每个长度 3):

```
h_我 = [1.0, 0.0, 0.0]
h_今 = [0.0, 1.0, 0.0]
h_天 = [0.0, 0.0, 1.0]
```

**这一步的"数字"只取决于"是哪个字",跟上下文无关。** 所以"我今天"里的"天"和"明天"里的"天",在这一步是完全一样的 `[0,0,1]`。

### 步骤 2:每层算 Q/K/V(就是拿"字的数字"去乘旋钮)

假设第 1 层的 `W_K` 旋钮(3×2)已被训练成:

```
W_K =
  [1, 0]
  [0, 1]
  [1, 1]
```

(意思是:3 维输入 → 2 维 K)

现算每个字的 K(= h × W_K):

```
K_我 = [1.0,0.0,0.0] × W_K = [1×1+0×0+0×1, 1×0+0×1+0×1] = [1.0, 0.0]
K_今 = [0.0,1.0,0.0] × W_K = [0.0, 1.0]
K_天 = [0.0,0.0,1.0] × W_K = [1.0, 1.0]
```

**看清楚:K 就是这么乘出来的。它不是存的,不是查的,是"字的数字 × 旋钮"当场算出来的。** V 同理(换成 `W_V`)。

### 步骤 3:attention——让每个字"只看前面",把别人的 V 混进来

"只看前面"靠一个 mask。对"我今天"三个位置:

```
       看 我  看 今  看 天
我:     ✓     ✗     ✗      ← 我只看自己
今:     ✓     ✓     ✗      ← 今看 我、今
天:     ✓     ✓     ✓      ← 天看 我、今、天
```

以"天"为例(它能看到全部三个字):

1. 算"天"的 Q 和每个能看到的 K 的相关度(点积):`score = Q_天 · K_我, Q_天 · K_今, Q_天 · K_天`
2. softmax 归一化成权重(加起来=1),比如得到 `[0.2, 0.3, 0.5]`
3. 用权重加权 V:`新h_天 = 0.2×V_我 + 0.3×V_今 + 0.5×V_天`

**这一步之后,"天"的数字串变了——它吸进了"我"和"今"的信息。** 这就是"上下文被混进来"。从这一刻起,这个"天"已经不只是"天"字本身,而是"我今天"语境下的"天"。

> 关键观察:在步骤 2,同一个"天"字不管放哪句话,K_天 都是 `[1,1]`(因为只乘固定的 h_天 和固定 W_K)。**但步骤 3 之后 h_天 变了**,如果有第 2 层,第 2 层再算 K_天 = 新h_天 × W_K(第2层),这才会因上下文不同而不同。**这就是"同字不同语境 K/V 不同,从第 2 层才涌现"的精确含义。**

### 步骤 4:过 FFN、最后一层,变回"每个词的概率"

简化:attention 输出再过个 FFN 旋钮,然后最后一层 `W_out`(3×5)把"天"位置的 3 维数字串变回 5 维(对应 5 个词),再 softmax 成概率。比如输出:

```
我:0.05  今:0.05  天:0.10  很:0.70  开:0.10
```

模型预测下一个字是"很"(概率 0.70 最高)。**这就是一次前向传播的终点。**

## 第 3 章:loss 到底怎么算出来的(你最问的"怎么量化差距")

接上,模型对"我今天"的下一个字给了概率分布:

```
很:0.70   ← 真值就是这个
其他四个加起来 0.30
```

真值是"很"。衡量"错多少"用**交叉熵损失**:

```
loss = -log(模型给真值"很"的概率)
     = -log(0.70)
     ≈ 0.357
```

- 如果模型给"很"的概率是 `1.0`(全押对了):`loss = -log(1.0) = 0` → 没错。
- 如果给"很"的概率是 `0.01`(几乎全猜错):`loss = -log(0.01) ≈ 4.6` → 很错。

**所以"差距多大"就是 `-log(给真值的概率)`。** 概率越高(loss 越接近 0)说明越对,概率越低(loss 越大)说明越错。这就是量化。一个 batch 里所有样本的 loss 取平均,就是这一趟的总 loss。

---

## 第 4 章:反向传播——怎么"往能让答案更对的方向调旋钮"

loss 是 `0.357`,表示"很"的概率还嫌低。目标:让这个概率升上去。

数学上(不展开推导,只讲直觉):**对每个旋钮算一个"偏导数"——意思是"这个旋钮往大调一点点,loss 会变大还是变小、变多少"。** 这个偏导数叫**梯度**。

- 某旋钮梯度是负的 → 把它调大一点,loss 会降 → 那就调大。
- 某旋钮梯度是正的 → 调小一点 loss 会降 → 那就调小。

更新公式(梯度下降):

```
新旋钮值 = 旧旋钮值 - 学习率 × 该旋钮的梯度
```

- `学习率` 是一个小步长(如 0.001),防止一步迈太大。
- 整件事就叫"反向传播算梯度 + 梯度下降调旋钮"。**所谓"反向",是从 loss 往前一层层算每个旋钮的梯度,方向和前向相反。**

调完一轮,旋钮变了一点点,模型对这条数据的 loss 降了一点点。重复几亿次,所有旋钮收敛到一组"比较能猜准"的值。

> 你问的"怎么判断差距多大"答案:**loss = -log(给真值的概率)**,一个具体的数。"怎么调"答案:**算每个旋钮的梯度,按 `新=旧-学习率×梯度` 微调。**

## 第 5 章:训完之后,到底什么固定了

训练停了之后:

| 类别 | 状态 |
|------|------|
| `E`, `W_K`, `W_V`, `W_Q`, `W_O`, `W_FF`, `W_out`, `W_norm` ... **所有旋钮** | **数值固定,存进 `.safetensors` 模型文件,永不再变** |
| Q, K, V | **不存在**于文件里,用的时候现算 |
| hidden state h(字的数字串) | 用的时候现算,随输入变 |

**所以"权重固定"=上面那堆旋钮的数值固定。** 没有别的。"K/V 固定"这种说法不存在——K/V 是用的时候 `h × W_K` 临时乘出来的。

## 第 6 章:再精确回答"字的数字"是什么、是否随上下文变

"字的数字"(h)分阶段看:

| 阶段 | "天"的 h 由什么决定 | 不同句子里"天"的 h 一样吗 |
|------|---------------------|--------------------------|
| 查 embedding(层 0) | 只由"天"的 id 决定 | **完全一样** |
| 第 1 层算 K/V | = h(层0) × W_K(层1) | **一样**(因为 h 一样、W_K 固定) |
| 第 1 层 attention 输出 | 把前文 V 混进来 | **不一样了** |
| 第 2 层算 K/V | = h(层1输出) × W_K(层2) | **不一样** |

**结论:"字的数字"在最开始(查表那步)是固定的、跟上下文无关;经过 attention 混入上下文后,从第 1 层 attention 输出开始,就随上下文不同而不同了。** K/V 同理——第 1 层算的时候还一样,第 2 层起不一样。

## 第 7 章:Prefill——输入 prompt 时发生什么

模型训好了(旋钮固定),你输入 prompt "我今天"。这段 prompt 全已知,处理它叫 **prefill**:

1. prompt 所有字一起并行塞进流水线(因为都已知,不用等)。
2. 每层:用固定的 `W_K`/`W_V` 把每个字的 K/V **现算出来**(就是第 2 章步骤 2 那套)。
3. attention 让每个字只看前面,把上下文混进来(步骤 3)。
4. 一路算到最后一层,出口给出"下一个字预测"。
5. **同时**:把每一层算出的每个字的 K/V,**抄一份存进 KV cache**。

> 你问"prefill 用固定 KV 还是啥"——答案是:**prefill 不用"存的 KV",而是用固定旋钮把 prompt 的 K/V 现算出来,然后把结果存进 cache。** 存进 cache 的这些 K/V,才是后面 decode 要复用的东西。

prefill 结束时:cache 里装着 = prompt 每个字 × 每层的 K/V,一份完整快照。

---

## 第 8 章:Decode——为什么要存 KV cache

prefill 猜出第一个新字"很"后,要继续猜下一个字,进入 **decode**(一个一个蹦字)。

每蹦一个新字(比如"很"),它要拿自己的 Q 去和**之前所有字("我今天")的 K、V** 做匹配(查字典)。这些历史 K/V 从哪来?

- **不存 cache**:每个新字来了,都得把"我今天"重新乘一遍旋钮重算 K/V → 重复劳动,慢。
- **存了 cache**:prefill 时已算好存进去了,直接取出来用,**不重算**。

这就是 KV cache 的意义。**KV cache 里存的东西 = 每个历史 token 在每一层算出的 K 和 V,用固定旋钮现算、存下来供后续 token 复用。** LMCache 把这个 cache 从 GPU 显存搬到 CPU/磁盘/远端,解决长上下文显存放不下的问题。

---

## 第 9 章:训练并行 vs 推理串行(彻底说清)

- **训练**:整句话来自数据集,全部已知 → 每层可以**所有位置一起算** Q/K/V 和 attention(一次大矩阵乘),mask 保证只看前面。并行的维度是"位置"。
- **推理 prefill**:prompt 全已知 → 和训练一样并行。
- **推理 decode**:下一个字未知,必须等上一个字猜出来才能当输入 → 本质串行,一个一个蹦。但每步里历史 K/V 可从 cache 取,不重算。

**并行/串行的根源是"输入是否已知",不是 attention 本身。**

## 第 10 章:全文一图收束

```
出厂:   所有旋钮(权重)= 随机数
  ↓ 训练(前向→算loss→反向算梯度→按梯度调旋钮,重复几亿次)
训好:   所有旋钮数值固定,存进 .safetensors        ← "权重固定"指这个
  ↓ 用它推理
prefill: prompt 全已知 → 并行 → 每层用固定旋钮把每个字的 K/V 现算 → 存进 cache → 出口猜下一字
  ↓
decode:  一个一个蹦字 → 每个新字拿 Q 去 cache 查历史 K/V(不重算)→ 猜下一字 → 它的 K/V append 进 cache
```

**被训练和固定的是"旋钮(权重)";K/V 是"输入 × 固定旋钮"现算出来的临时产物;prefill 现算并存 cache,decode 取出复用避免重算。** 这一句是全文的纲,其余都是它的展开。
